# 更新记录

## Unreleased

## v0.2.1 - 2026-07-23

- 为标准和重型任务新增轻量 `run_record`：用同一组节点编号对齐施工计划与实际执行。
- 升级 Hs Feedback：优先读取运行证据定位首次偏离，经确认后回写，并从最早受影响节点重跑验证。
- 新增回归样本契约与模板，区分私有业务缺口、通用方法缺陷、偶发执行错误和用户临时偏好。
- 明确 Hs 不自主改写真源；通用 Skill 修改必须绑定回归样本，未验证不得宣称问题已解决。

## v0.2.0 - 2026-07-21

- 在 Hs Analysis 中新增“损失来源、过程断点、根因机制”三层证据门槛。
- 当根因证据不足时，强制降级为候选假设，并输出替代解释、判断边界和待补数据。
- 新增全渠道零售完整 Demo，包括 8 份 Source、固定种子合成数据、5 类任务、施工图和标准答案。
- 新增独立 Demo 工作区初始化、重置、隐私扫描和可复现性校验。
- 将私有业务拦截词迁移为本地 denylist，公开仓库不再保存维护者的公司或项目名称。
- 采用 Apache License 2.0，并明确免费 Skill 与咨询服务的边界。

## v0.1.2 - 2026-07-12

- 在 README 与架构文档中新增 Hs Skill 协作总览图，明确入口、建图、内部分析、外部研究、审计、建表、输出与反馈回写之间的关系。

## v0.1.1 - 2026-07-12

- 明确 Hs 的核心交付是平台无关的 Markdown 方法、业务图谱和数据资产协议。
- 新增多平台接入说明：文件型 Agent、项目知识库型 AI 与单次对话型 AI 可按不同能力边界使用 Hs。
- 将 Codex 一键同步、命名调用和界面元数据定义为可选适配器，不再作为默认安装前提。

## v0.1.0 - 2026-07-12

首个可分发版本。

- 提供 Hs Entry、Onboarding、Graph、Analysis、Research、Output、Feedback 能力。
- 增加数据型任务发布链：数据契约、指标审计、三层表格构建。
- 提供可复制的双边业务演示图谱与安全示例数据。
- 提供本机同步、Demo 初始化与基础产品校验脚本。
