# 数据与隐私边界

Hs 可以帮助你读取、组织和分析本地业务资料，但不会替你判断一份公司数据是否适合公开。

## 不要提交到 GitHub 的内容

- 原始业务数据、客户信息、个人信息、账号密钥、访问令牌。
- 私有 SQL、内部链接、未公开财务或经营资料。
- 含有可识别公司、个人或客户信息的业务图谱与输出报告。

仓库默认忽略 `business_graphs/`、`outputs/`、`private/` 和 `.obsidian/`。请把真实业务资产保留在本地私有工作区，而不是提交到仓库、公开 Issue 或讨论区。

## 本地私有词拦截

复制根目录的 `private-terms.example`，在本机创建 `private/private-terms.txt`，每行写一个不允许进入公开包的公司名、项目名、内部缩写或个人标识。`private/` 已被 Git 忽略，运行 `scripts/validate-package.py` 时会自动载入这份本地词表，但词表本身不会进入公开仓库。

也可以通过环境变量 `HS_PRIVATE_TERMS_FILE` 指向其他本地词表。

## 提交问题时

请优先提供脱敏后的字段名、虚构样本、报错信息或最小复现步骤。不要把原始数据表直接贴到公开讨论中。

## 发现敏感信息泄露

请不要创建公开 Issue。先联系仓库维护者，并在修复前避免传播相关链接或截图。
