# Demo 演练：从业务图谱到可交付结论

这份演练用于验证 Hs 的完整数据任务发布链，不用于说明任何真实行业。

## 1. 初始化 Demo

在仓库根目录运行：

```bash
./scripts/bootstrap-demo.sh
```

它会把安全的双边撮合示例业务复制到当前目录的 `business_graphs/`，其中包含指标树、Source 卡与 2026 年 1 至 6 月的模拟数据。

## 2. 从入口发起任务

在你接入 Hs 的 AI 环境中输入：

```text
请使用 Hs 的 hs-entry 工作流，读取 demo-two-sided-market，分析 2026 年 5 月收入变化的主要原因，并给我一份可汇报的简报。
```

在 Codex 中，也可简写为 `$hs-entry` 加同一段任务。其他环境按[平台适配说明](platform-adapters.md)提供 `hs-entry` 与相关图谱文件即可。

预期行为：

1. `hs-entry` 读取 registry、manifest、指标索引、资产索引和相关 Source 卡。
2. 它展示施工图，写清范围、数据源、交付物、检查点和暂不覆盖的内容。
3. 它停下，等待你确认施工图。

## 3. 确认后应经过的发布链

确认施工图后，预期路径为：

```text
hs-data-contract
-> hs-analysis
-> hs-metric-audit
-> hs-table-builder
-> hs-output
```

你应该能看到：

- 一份数据契约：时间范围、月份 × 供给类型粒度、收入公式、数据文件和假设。
- 一份临时指标审计表：收入、意向用户数、有效匹配数、匹配转化率、单次匹配收入等为什么纳入或排除。
- 一份审计日志：同比/环比、分子分母和收入回算是否通过。
- 一份简报：结论、关键图表或表格、口径、数据截止日期和边界。

## 4. 故意制造一个 Bad Case

把 Source 卡中的数据文件路径改错，或删除一个字段后再次运行同一任务。

预期行为：

- `hs-data-contract` 应把任务标记为 `blocked` 或 `conditional`。
- 不应继续用猜测补齐数据。
- 需要修复图谱或规则时，应转入 `hs-feedback`，再由 `hs-graph` 维护资产。

## 5. 验收标准

Demo 跑通不等于 Hs 已适配你的业务。它只证明：

- 安装与调用路径正确；
- 施工图不会只靠自然语言猜测；
- 数据任务不会跳过口径、审计和三层表格；
- 出错时系统能停下并指向正确的修复位置。
