# 完整演示环境建设施工图

## Task

- 任务：建设一套可公开、可复现、与任何真实企业隔离的 Hs 完整演示环境。
- demo_id：`demo-omnichannel-retail`
- 虚拟业务：全渠道自营电商零售。
- 目标：支撑 Hs、业务图谱、指标树、数据分析、审计、建表、输出和反馈回写的长期演示。
- 交付对象：Hs 使用者、潜在付费用户、企业经营者和公开内容观众。
- 任务强度：重型。

## 用户已确认的决策

1. 不使用招聘业务，也不制作任何真实业务的脱敏复刻。
2. 采用全渠道电商零售作为虚拟业务。
3. 同时建设 GitHub 可公开真源与独立演示工作区。
4. 第一版按约 30 个月、6 个城市、约 30 个指标设计，遇到问题再调整。
5. 第一版优先覆盖内部分析全链路；`hs-research` 预留接口，后续单独补充实时外部研究演示。

## Scope Gate

### 本轮覆盖

- 一套完全虚构的业务模式、经营机制和组织边界。
- 指标、维度、层级关系、业务事实、Source 卡、绑定关系和索引。
- 可复现的合成数据源及明确的标准答案。
- 五类首批演示命题。
- GitHub 公开真源与本地可重置演示工作区。
- 隐私扫描、指标闭环、Source 绑定和演示回归校验。

### 本轮不覆盖

- 任何真实企业、真实用户、真实客户或真实组织的数据。
- 招聘、供应链、旅游等与维护者真实履历高度重合的业务复刻。
- 实时互联网研究结果的固定答案。
- 为了视觉效果而提前制作大量报告、网页或视频素材。
- 在业务机制和标准答案确认前生成大批随机数据。

## 安全边界

- 不读取或复制工作区现有私有业务图谱来生成本 Demo。
- 不使用真实公司名、组织名、字段名、专属指标、历史事件和数值。
- 所有资产必须标记为 `synthetic` 或 `demo`。
- 所有面向用户的输出必须声明“虚构数据，仅用于演示”。
- 数据由固定随机种子生成，生成规则和结果均可复现。
- 独立演示工作区不得链接、挂载或引用当前私有工作区路径。
- 隐私扫描必须覆盖 Markdown、CSV、JSON、HTML 和 Excel 可读内容。

## 首批演示命题

1. 某月收入为什么下滑，主因和次因分别是什么？
2. 哪个区域综合表现最好，优势来自需求、供给还是效率？
3. 提高下一季度收入目标，需要哪些过程指标承担压力？
4. 新增一个商品品类后，如何更新业务图谱、指标和 Source 绑定？
5. 当数据源路径、字段或口径发生错误时，Hs 能否停止任务并完成 Feedback 回写？

## 交付结构

### GitHub 可公开真源

- 完整虚拟业务图谱。
- 合成数据与可复现生成规则。
- 五类演示任务及标准答案。
- 初始化、重置、校验脚本和使用说明。

### 独立演示工作区

- 从公开真源初始化，不手工复制私有资产。
- 用于日常试验、录屏、截图和生成可分享产物。
- 输出物与临时修改不反向污染 GitHub 真源。

## Checkpoints

| 检查点 | 检查什么 | 不通过时怎么办 |
|---|---|---|
| `C01` 业务边界 | 是否独立于真实履历和真实企业 | 更换业务机制或删除相似资产 |
| `C02` 标准答案 | 每个命题是否有预先设计的证据链 | 暂停数据生成，先补答案设计 |
| `C03` 指标闭环 | 结构拆解、公式拆解、条件拆解是否一致 | 返回图谱骨架修正 |
| `C04` 数据闭环 | 父子加总、分子分母和跨表映射是否成立 | 返回生成规则修正，不用兜底公式掩盖 |
| `C05` 范围命中 | 分析是否覆盖预期区域、品类、渠道和时间 | 降级结论或修正 Source/HIER |
| `C06` 隐私安全 | 是否含真实名称、路径、字段或历史数据 | 阻断发布并清理全部关联资产 |
| `C07` 可复现性 | 是否能从空工作区初始化并得到相同结果 | 修复初始化脚本和固定种子 |
