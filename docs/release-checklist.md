# 发布检查清单

这份清单用于记录 Hs 在公开发布或付费交付前，还需要补齐什么。

## 必须具备

- [x] 产品包是 Hs 的唯一产品真源。
- [x] 至少一个参考运行环境能从产品包接入并完整执行 Demo；Codex 一键同步只是其中一个适配器。
- [x] 所有 Hs 技能都有合法的 `SKILL.md` frontmatter。
- [x] 可复用技能规则中不出现私有业务名称、数据、路径或案例。
- [x] 空业务图谱模板足够支撑 onboarding。
- [x] 演示业务图谱只使用公开安全或虚构数据。
- [x] 演示 registry、Source 卡、数据文件和 Markdown 链接都能通过校验。
- [x] 完整 Demo 覆盖施工图、数据契约、分析、审计、建表和输出链路。
- [x] 快速开始流程能在没有私有上下文的情况下初始化可运行 Demo。
- [x] `hs-entry` 能在标准/重型任务前生成施工图。
- [x] 标准/重型数据任务的施工图明确数据契约、分析与中间测算、指标审计、建表和交付节点。
- [x] 数据契约未 `ready` 或 `conditional` 时，不开始复杂计算。
- [x] 指标审计存在未解决 `fail` 时，不生成可直接汇报的最终表或报告。
- [x] 最终呈现层只引用审计通过的中间结果，不承担隐藏的复杂计算。
- [x] `hs-graph` 定义新增、更新、链接、废弃资产及同步索引的工作流。
- [x] `hs-feedback` 定义 Bad Case 分类与真源回写路由。
- [x] 已确定仓库公开范围、Apache-2.0 许可证和敏感数据处理方式。
- [x] 已准备 `v0.2.0` 版本号、更新记录和 GitHub remote。

## 重要但可以稍后

- [ ] 增加断链和索引过期检查脚本。
- [ ] 增加一个更精致的演示产出物。
- [ ] 增加常见问题 FAQ。
- [ ] 确定分发方式：压缩包、GitHub 仓库、私域售卖包或安装脚本。
- [ ] 确定真实用户拿到的是示例，还是一棵私有 starter graph。

## 当前已知缺口

- 当前校验已覆盖本地 Markdown 断链，但尚未实现全部索引内容与资产状态的语义一致性检查。
- Codex 技能通过脚本同步，不是实时软链接；其他平台仍需各自的项目知识库或文件接入方式。
- Feedback/回写技能已定义工作流，但还需要更多真实用户 Bad Case 验证其稳定性。
- 完整 Demo 可以确定性复现数据和标准答案，但跨 AI 环境的最终表达质量仍会存在差异。
