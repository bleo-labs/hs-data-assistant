## 基础量与派生量闭合原则

### 先生成的基础量

- 用户漏斗人数: 访问、详情、加购、提交、支付、新支付、老支付。
- 交易原子量: 支付订单、每单件数、SKU 成交价、销售件数和退款交易金额。
- 供给状态: 在售 SKU、有货 SKU。
- 成本与费用: 商品成本、营销费用、履约费用、渠道费用。
- 履约分子分母: 已完成订单、准时完成订单。
- 归因分子分母: 可归因营销费用、可归因新支付客户、可归因净销售额。

### 只通过公式生成的派生指标

- GMV、净销售额、毛利润、贡献利润。
- 毛利率、贡献利润率、全部漏斗转化率。
- 有货率、购买频次、客单价、平均销售单价、退款率、准时履约率。
- 新客获客成本、付费投放产出比、老客占比。

## 必须通过的闭合关系

```text
支付客户数 = 新支付客户数 + 老支付客户数
GMV = 支付订单数 × 客单价
GMV = 销售件数 × 平均销售单价
净销售额 = GMV - 退款金额
毛利润 = 净销售额 - 商品成本
贡献利润 = 毛利润 - 营销费用 - 履约费用 - 渠道费用

访问用户数 >= 商品详情用户数 >= 加购用户数
>= 提交订单用户数 >= 支付客户数

有货SKU数 <= 在售SKU数
准时完成订单数 <= 已完成订单数
```

其中 GMV 的生成顺序为：先生成支付订单与订单内商品件数，再按 SKU 成交价汇总商品成交金额；客单价和平均销售单价是校验用派生结果，不参与反向随机生成。

## Source 边界修正

原“流量漏斗月表”无法同时支持渠道流量质量和品类供给冲击分析，已拆成：

- `SRC-0001` 渠道流量漏斗月表。
- `SRC-0002` 商品行为漏斗月表。

商品行为漏斗使用“每位月度用户一个主浏览品类”的合成口径，使品类基础量可以闭合到渠道总量。这个规则只服务于公开 Demo 的可审计性，不假设真实电商天然如此。
