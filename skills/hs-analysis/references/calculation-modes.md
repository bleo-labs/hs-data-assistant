# Calculation Modes

测算层先分两类：描述性统计和关系型测算。

## Mode 1: 描述性统计

不利用指标之间的相关关系产生新计算，只按照指标树和数据源展开事实。

回答：

- 现在是多少
- 同比/环比是多少
- 哪个业务线高或低
- 哪个指标变好或变差
- 和大盘、历史、目标相比如何

适用：

- 经营看板
- 事实复盘
- 奖励证明
- 述职材料
- 初步诊断

产物：

- 指标对比表
- 分业务线看板
- 证据表
- 异常清单

门禁：

- 必须先确认对比口径。
- 不解释因果，只呈现事实和可支持的对比。
- 不把相关关系写成测算公式。
- 仍然必须沿指标树展开到过程指标；描述性统计不是只做一级贡献拆解。
- 如果过程指标没有数据，只能输出“待验证的过程指标表”，不能把结果层贡献写成完整原因。

## Mode 2: 关系型测算

利用指标之间的公式关系、明确业务机制或经验区间，产生预测、归因、压力反推或上下限。

回答：

- 未来大概是多少
- 哪个变量影响更大
- 目标上浮后压力落在哪
- 指标上下限由哪些变量决定
- 关键资源不变时效率要提升多少

适用：

- 预算
- 目标制定
- 方案论证
- 压力测试
- 经营模型

产物：

- 测算模型
- 贡献拆解
- 情景预测
- 目标反算
- 上下限区间

关系分级：

1. 公式关系：可以直接进入计算。
2. 明确业务机制：可以进入测算，但必须标注假设。
3. 经验相关关系：只能用于区间、校验、解释，不能冒充因果。

门禁：

- 先确认哪些指标可以直接求和，哪些指标必须由基础指标重算。
- 先确认哪些字段只是展示维度，不能直接当作原因。
- 先把资源变化和效率变化拆开看，不要混成一句解释。
- 固定或已知的资源项不机械套同比。
- 率类指标必须用分子分母重算。
- 预测粒度必须选择稳定层级。
