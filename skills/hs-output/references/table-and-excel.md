# Table And Excel Standard

这份文件约束 Hs 输出表格、Excel 看板和测算表的可读性与可复查性。

## 1. 表格分层

重型任务的 Excel 优先拆成三类表：

- 数据源层：原始数据或清洗后的可复用数据。
- 中间测算层：公式、拆解、预测、校验。
- 最终呈现层：给读者看的结论、关键输入和关键过程指标。

不要让最终呈现层承担复杂公式。复杂公式应放在中间测算层，最终层只引用结果。

## 2. Sheet 命名

推荐顺序：

1. `00_使用说明`
2. `01_最终呈现`
3. `02_关键结论`
4. `10_中间测算_...`
5. `20_校验_...`
6. `90_数据源_...`

若用户已有命名习惯，可沿用；但必须让最终呈现、测算、数据源一眼可区分。

## 3. 表内结构

一张表必须能回答：

- 这张表解释哪个主题指标？
- 行维度是什么？
- 列指标是什么？
- 哪些是输入？
- 哪些是计算结果？
- 哪些是风险或备注？

推荐字段顺序：

1. 维度字段
2. 历史表现
3. 当前/预测/目标值
4. 同比、环比、目标差
5. 关键过程指标
6. 解释、风险、来源

## 4. 汇总与明细

- 汇总行放在对应明细集合的上方，便于先看结论。
- 可加指标从明细求和。
- 率类、人效、ARPU、亩产、转化率必须由分子分母重算，不能直接平均或求和。
- 空层级必须保留为“未展开”或等价占位，不能在透视或分组中丢行。

## 5. 颜色规则

颜色用于降低理解成本，不用于装饰。

建议：

- 深色：只用于文件标题区或大模块标题。
- 浅蓝：历史表现或只读计算结果。
- 浅黄：用户可输入或可调整项。
- 浅绿：预测、达成、正向表现。
- 浅红：风险、差距、异常。
- 灰色：辅助说明、不可用、不适用。

关键要求：

- 不要让深色汇总行覆盖红绿条件格式。
- 输入项必须和计算项有明显区分。
- 同一组指标与对应同比/环比使用相近色系。

## 6. 数字格式

- 金额类必须标单位：元、万元、千元等。
- 比率类显示百分比。
- 比率变化统一用 PP。
- 大数字按交付对象选择单位，避免长串数字。
- 0 与空白不能混用：0 是明确为零，空白是缺失、不适用或不可用。

## 7. 公式与可复查性

- 关键结果必须能追溯到数据源或中间测算表。
- 不用 `IFERROR` 掩盖错误；如使用，必须保留错误检查列或说明。
- 不保留无用外链、循环引用、隐藏关键公式。
- 交付前检查汇总是否等于明细、分项是否等于总项、率类是否重算。

## 8. 最终呈现层

最终呈现层只保留读者需要看的内容：

- 核心结论。
- 关键输入。
- 关键结果。
- 关键过程指标压力。
- 风险提示。
- 数据截止时间和口径说明。

不要把完整中间测算、全部字段和临时校验都堆到最终呈现层。
